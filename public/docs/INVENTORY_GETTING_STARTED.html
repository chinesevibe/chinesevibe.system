คู่มือเริ่มต้นใช้งาน Inventory

เอกสารนี้สรุปการใช้งานโมดูลคลังสินค้าใน CNV WorkHub สำหรับผู้ดูแลระบบ, HR, Inventory, ผู้จัดการสาขา และทีมปฏิบัติการที่ต้องเริ่มใช้งานคลังแบบหลายสาขาให้เป็นระบบเดียวกัน

อัปเดตตามโครงสร้างระบบปัจจุบัน เปิด Inventory Workspace ↗ เปิด Dashboard ↗
ภาพรวม สิทธิ์เข้าใช้ ลำดับตั้งค่า เมนูหลัก งานประจำวัน ข้อมูลหลัก รายงาน ข้อควรระวัง Checklist เริ่มต้น

1. ภาพรวม Inventory ในระบบนี้

โมดูล Inventory ในระบบนี้ออกแบบสำหรับร้านอาหารและหลายสาขา โดยแยกการทำงานเป็น 2 ชั้น:

ชั้นงานปฏิบัติการ

รับเข้า, ใบเบิก, โอนสินค้า, บันทึกใช้จริง, แจ้งเสียหาย, นับสต็อก, ติดตาม alert และรายงานการเคลื่อนไหว

ชั้นข้อมูลหลัก

SKU, Supplier, สาขา inventory, คลังสินค้า และ BOM ซึ่งเป็นฐานข้อมูลให้ทุก flow ใช้งานร่วมกัน

แนวคิดหลัก: เริ่มจากข้อมูลหลักให้ถูกก่อน แล้วค่อยใช้งาน flow รับเข้า → เบิก → โอน → ใช้จริง → ตรวจนับ → รายงาน

2. ใครควรใช้เมนูไหน

บทบาท งานที่ควรทำ หมายเหตุ
Inventory / Dev / CEO ใช้งานได้ครบทั้งงานปฏิบัติการและข้อมูลหลัก เหมาะกับผู้คุมระบบคลังรวม
HR / ผู้ดูแลระบบ ดูภาพรวม, รายงาน, alert, master data ที่เกี่ยวข้อง ใช้ดูการควบคุมและตรวจสอบ
Branch Manager / พนักงานสาขา ใบเบิก, รับเข้าเฉพาะที่เกี่ยวข้อง, บันทึกใช้จริง, ตรวจนับ ไม่ควรเริ่มจากแก้ master data โดยไม่มีผู้คุมคลังกลาง

เมนู Inventory ถูกแยกสิทธิ์สำหรับงานปฏิบัติการกับข้อมูลหลักอยู่แล้ว ถ้าบางหน้ามองไม่เห็น มักเกิดจาก role ที่ไม่เปิดสิทธิ์ส่วนนั้น

3. ลำดับตั้งค่าก่อนเริ่มใช้งานจริง

ถ้ายังไม่เคยใช้โมดูลนี้ ให้เริ่มตามลำดับนี้เสมอ:

  1. สร้างหรือทบทวน สาขา inventory ให้ครบ
  2. สร้าง คลังสินค้า ของแต่ละสาขา เช่น Main, Sub, Receiving
  3. สร้าง Supplier ที่ใช้งานจริง
  4. สร้าง SKU / สินค้า / วัตถุดิบ พร้อมหน่วยนับและ min/max stock
  5. ถ้ามีการแปรรูปหรือสูตร ให้ตั้ง BOM
  6. เริ่มรับเข้าสินค้าล็อตแรก เพื่อให้ stock on hand มีฐานเริ่มต้น
อย่าเริ่มจาก “เบิก” หรือ “โอน” ก่อนที่ SKU, warehouse และ stock ตั้งต้นจะครบ เพราะจะทำให้รายงานและ stock movement เพี้ยนตั้งแต่วันแรก

/admin/inventory หน้าเริ่มต้นของคลัง ดูงานด่วนวันนี้และเข้า flow หลักจากจุดเดียว ทุกคนที่ทำงาน inventory
/admin/inventory/dashboard ดูภาพรวมเชิงบริหาร แยกตามสาขา คลัง และ alert ผู้จัดการ, HR, Inventory lead, CEO
/admin/inventory/stock ดู stock on hand, lot, expiry, สถานะคงเหลือ คลังกลาง, สาขา, ผู้ตรวจสอบ
/admin/inventory/inbound สร้างใบรับเข้า, รับสินค้าเข้าคลัง, บันทึก lot และจำนวน ฝ่ายคลัง / ฝ่ายรับสินค้า
/admin/inventory/requisition สร้างและติดตามใบเบิกจากสาขาหรือครัว สาขา, ครัว, Inventory
/admin/inventory/transfer โอนสินค้าระหว่างคลังหรือระหว่างสาขา คลังกลาง, ผู้จัดการสาขา
/admin/inventory/consumption บันทึกการใช้จริงและตัด stock ตามเหตุผลที่กำหนด ครัว, ผู้จัดการสาขา, Inventory
/admin/inventory/damage บันทึกของเสีย, แตกหัก, หมดอายุ หรือเสียหาย คลัง, สาขา, ผู้ตรวจสอบ
/admin/inventory/stock-count ตรวจนับ stock จริงและเปรียบเทียบกับระบบ คลัง, สาขา, Audit
/admin/inventory/reports ดูรายงาน stock, movement, variance, inbound, requisition, transfer, damage, audit trail ผู้คุมงานและผู้บริหาร

5. วิธีใช้งานแบบวันต่อวัน

5.1 เปิดวันทำงาน

  1. เข้า /admin/inventory หรือ /admin/inventory/dashboard
  2. ดู alert ว่ามี low stock, expiry, หรือรายการค้างรับ/ค้างโอนหรือไม่
  3. เลือกสาขาหรือคลังที่ต้องดูเป็นพิเศษก่อนเริ่มงาน

5.2 รับสินค้าเข้า

  1. สร้างใบรับเข้าที่ /admin/inventory/inbound/new
  2. เลือก supplier และคลังรับเข้า
  3. เพิ่มรายการสินค้า, จำนวน, หน่วย, lot และ expiry ถ้ามี
  4. บันทึกรับเข้า แล้วกลับมาตรวจ stock ว่าจำนวนเข้าแล้วจริง

5.3 เบิกสินค้าให้ครัวหรือสาขา

  1. สร้างใบเบิกที่ /admin/inventory/requisition/create
  2. เลือกสาขา/คลังที่ขอเบิก
  3. เพิ่มรายการและจำนวนที่ต้องการ
  4. ผู้เกี่ยวข้องตรวจสถานะว่าเป็น draft, pending, issued หรือ completed

5.4 โอนสินค้าระหว่างคลังหรือสาขา

  1. เข้า /admin/inventory/transfer/create
  2. เลือกคลังต้นทางและปลายทางให้ถูก
  3. เลือก lot ให้ชัดเมื่อสินค้ามีหลาย lot หรือมี expiry ต่างกัน
  4. ยืนยันโอนและติดตามรายการที่ปลายทางต้องรับให้ครบ

5.5 บันทึกใช้จริง / ของเสีย / ตรวจนับ

Consumption

ใช้เมื่อสินค้าถูกใช้งานจริงและต้องตัด stock ตามเหตุผลที่ระบุ

Damage

ใช้เมื่อสินค้าหมดอายุ ชำรุด เสียหาย หรือใช้ต่อไม่ได้

Stock Count

ใช้เมื่อมีการตรวจนับจริงเพื่อตรวจหาส่วนต่างและปิดรอบตรวจนับ

6. ข้อมูลหลักที่ต้องดูแล

6.1 SKU

เมนู /admin/inventory/sku ใช้เก็บรายการสินค้า/วัตถุดิบทั้งหมด ควรมี code, ชื่อ, หน่วยนับ, min/max stock และสถานะใช้งาน

6.2 Supplier

เมนู /admin/inventory/suppliers ใช้เก็บผู้จำหน่าย, ข้อมูลติดต่อ และสถานะการใช้งาน เพื่ออ้างอิงตอนรับเข้า

6.3 สาขา inventory และคลังสินค้า

เมนู /admin/inventory/branches และ /admin/inventory/warehouses ต้องตั้งให้ตรงกับโครงสร้างจริง เช่น สาขา A มีคลัง Main, Receiving, Kitchen เป็นต้น

6.4 BOM

เมนู /admin/inventory/bom ใช้เมื่อสินค้าหนึ่งรายการถูกสร้างจากหลายวัตถุดิบ เช่น เมนูอาหารหรือชุดประกอบ เพื่อรองรับการคำนวณการใช้จริงในรอบถัดไป

7. รายงานที่ควรใช้ประจำ

รายงาน เอาไว้ตอบคำถามอะไร
Stock On Hand ตอนนี้ของคงเหลือจริงในแต่ละคลังเท่าไร
Inbound ช่วงนี้รับของอะไรเข้าเท่าไร จาก supplier ไหน
Requisition สาขาหรือครัวเบิกอะไรไปบ้าง และค้างสถานะอะไร
Transfer มีของถูกโอนจากคลังไหนไปคลังไหน และปลายทางรับครบหรือยัง
Consumption ใช้ของอะไรไปเท่าไร และตัด stock ด้วยเหตุผลใด
Damage ของเสียเกิดจากอะไร จุดไหนเสียบ่อย และมูลค่าความเสียหายเป็นเท่าไร
Variance / Audit Trail ระบบกับของจริงต่างกันตรงไหน และใครเป็นคนแก้ไขรายการ

จุดเริ่มต้นของรายงานอยู่ที่ /admin/inventory/reports

8. ข้อผิดพลาดที่มักเจอเมื่อเริ่มใช้

เลือกคลังผิด

ทำให้รับเข้า, โอน, หรือบันทึกใช้ไปลง warehouse ผิด และรายงานคงเหลือเพี้ยน

สร้าง SKU ซ้ำ

ของชนิดเดียวกันแต่คนละ code ทำให้รวมยอดและตรวจนับลำบาก

ไม่กำหนด lot / expiry

ถ้าสินค้ามีวันหมดอายุ จะตาม FEFO และ expiry alert ได้ไม่เต็ม

ตัด stock ข้าม flow

ใช้ damage หรือ consumption ผิดบริบท จะทำให้ audit trail อ่านยาก

ถ้าเป้าหมายคือ “ใช้งานให้ถูกตั้งแต่วันแรก” ให้มีคนกลาง 1 คนตรวจว่า branch, warehouse, supplier, sku และหน่วยนับถูกต้องก่อนเปิดให้หลายคนใช้งานพร้อมกัน

9. Checklist สำหรับเริ่มใช้จริง

เมื่อ checklist นี้ครบ ระบบ inventory จะพร้อมสำหรับการ demo, handoff หรือเริ่มใช้จริงแบบมีวินัยมากขึ้น
