Design Notes

UX rule

สิ่งที่อยู่ในกรอบกะ

ควรมีเฉพาะคนที่เกี่ยวกับการทำงานจริงของกะนั้น และ lane หลักควรโชว์เฉพาะคนที่ยังเปิดรอบอยู่

สิ่งที่ไม่ควรปน

ลา ขาด วันหยุด และยังไม่ถึงเวลาเริ่มกะ ไม่ควรแทรกใน focus list เพราะทำให้การสแกนคนที่ยังอยู่ในงานช้าลง

History: Audit Ledger

หน้า history ควรเน้น flow “กรองช่วงเวลา → เห็นสรุป → เห็น anomaly → ลงลึกเป็นรายการ” มากกว่าจะเริ่มจากตารางล้วน
