# UI Implementation Skill

## Purpose
Implement clean, production-ready UI without breaking existing business logic.

## Rules
1. Reuse existing components, layouts, Tailwind classes, shadcn/ui patterns, and design tokens.
2. Do not invent a new design system if the repo already has one.
3. Do not change database schema, API contracts, auth logic, or business logic unless explicitly requested.
4. Keep route behavior intact.
5. Avoid new dependencies unless absolutely necessary.
6. Separate UI structure from data/calculation logic.
7. Preserve existing Thai/English wording unless improving obvious UI clarity.
8. Improve layout hierarchy, spacing, readability, and primary action visibility.
9. Include loading, empty, and error states if they already exist.
10. Make the UI responsive by default.

## Output
Report files changed, UI/UX improvements, assumptions, validation results, and remaining risks.
