# Dashboard Table UX Skill

## Purpose
Improve data-heavy dashboards and tables so admins can scan, filter, understand, and act faster.

## Core Rules
1. Show important data first.
2. Limit columns to what helps decision-making.
3. Use clear status badges.
4. Make filters easy to understand and reset.
5. Separate primary actions from secondary actions.
6. Use pagination or limits for growing data.
7. Provide loading, empty, and error states.
8. Make row actions predictable.
9. Avoid hiding critical operational issues.
10. Keep mobile layout usable.

## Employee Profile Attendance Priority
For employee attendance detail pages, prioritize:
- Employee context
- Selected date
- Shift context
- Check-in/check-out
- Missing check-in/out
- Cross-day shift context
- Correction actions
- History as secondary information

## Output
Report problems found, UX improvements made, table/section changes, filter/search/pagination changes, and remaining risks.
