1. ภาพรวมสถาปัตยกรรม

ระบบนี้ใช้ HR App (Next.js + Supabase) เป็นศูนย์กลางข้อมูลพนักงาน พนักงานใช้งานผ่าน LINE ส่วน Odoo 19 เป็นเครื่องมือคำนวณเงินเดือน

ทิศทางข้อมูล Payroll: HR App push ไป Odoo ผ่าน XML-RPC — Odoo ไม่ดึงข้อมูลจากเว็บเอง

2. ลงทะเบียนและเข้าระบบ

2.1 พนักงานใหม่ (Self-register)

LINE Login → /register → กรอกชื่อ/เบอร์/สาขา → สร้าง hr_employees (รออนุมัติ) → แจ้ง HR ผ่าน LINE Group → HR อนุมัติใน Admin → status: active → ได้ Rich Menu LINE
  • Auth: LINE Login → Supabase session (JWT มี line_user_id)
  • สาขา Head Office (รหัส 000) → default pay_type = monthly
  • สาขาอื่น → default pay_type = hourly

2.2 HR สร้างพนักงานเอง

/admin/employees/new — กรอกโปรไฟล์ครบ: ชื่อ, สาขา, pay_type, เงินเดือน/ชม., สัญชาติ, วันจ่าย, บัญชีธนาคาร

2.3 Login ตาม Role

Roleเข้าได้
employeeLINE / Portal
branch_managerAdmin สาขา
hrAdmin เต็ม + อนุมัติทุกอย่าง
admin, ceo, devAdmin ตามสิทธิ์

3. งานประจำวันของพนักงาน (LINE)

Rich Menu: เช็คอิน · OT · เอกสาร · คลัง · ร้องเรียน · ติดต่อ HR

3.1 เช็คอิน / เช็คเอาท์

LINE แชร์ตำแหน่ง หรือ QR (/liff/checkin) → ตรวจ geofence 200m จากสาขา (ถ้าเปิด) → บันทึก hr_attendance → คำนวณ late / work_hours

3.2 ยื่น OT

/liff/overtime → กรอกวันที่/เวลา/เหตุผล → status: pending_hr → แจ้ง HR LINE Group → HR อนุมัติที่ /admin/overtime → approved → นับชม. OT ตอน payroll

กฎปัจจุบัน อนุมัติโดย HR Officer (role = hr) เท่านั้น — ไม่ผ่าน Branch Manager

3.3 ลา (Leave)

/liff/leave หรือ Portal → pending_hr → HR อนุมัติ /admin/leaves → ชม.ลา (sick / annual) ไป payroll

3.4 Attendance รายวัน

พนักงาน submit ชม. → pending_hr → HR อนุมัติ → ชม.เข้า hr_payroll_hour_lines (ledger) — ใช้กับพนักงาน hourly

3.5 อื่นๆ

  • เอกสาร — ยื่น/อนุมัติเอกสาร HR
  • ร้องเรียน — ส่ง complaint
  • Portal (/portal/*) — ดู attendance, ลา, ประกาศ

5. Payroll Workflow

5.1 ประเภทการจ่าย

pay_typeใครฟิลด์ salaryชม.ที่ sync
monthly Office (สาขา 000) เงินเดือน/เดือน OT + ลา (base จาก wage)
hourly หน้าร้าน บาท/ชม. ชม.จาก ledger ที่อนุมัติแล้ว

5.2 สิ้นเดือน — ใน HR App

  1. ปิด OT / ลา / attendance — อนุมัติให้ครบ
  2. /admin/payroll — ดูสรุปชม.เดือนนั้น
  3. /admin/payroll/odoo:
    • Sync รายชื่อพนักงาน → Odoo (โปรไฟล์ + wage + structure)
    • Sync Payroll เลือกเดือน (payslip + batch)
  4. /admin/payroll/settings — SSO, OT multiplier, ชื่อ Odoo structure

6. Odoo Sync (Push)

ลำดับการ Sync Payroll

HR (กด Sync) │ ▼ POST period YYYY-MM sync-payroll API │ ├──▶ Supabase: พนักงาน + OT/ลา/ledger │ └──▶ loop แต่ละคน: ├── hr.employee + tag Pay-04/05 ├── hr.version wage + structure └── hr.payslip + compute │ ▼ Payslip Batch แยก วันที่ 4 vs 5 │ ▼ สรุปผล → HR
Odoo 19: ใช้ hr.version แทน hr.contract เก่า

8. Flow ครบวงจร (หนึ่งเดือน)

[HR สร้าง/อนุมัติพนักงาน] ──▶ [พนักงานเช็คอินทุกวัน] ──▶ [ยื่น OT/ลา] │ ▼ [โอนเงิน วันที่ 4 หรือ 5] ◀── [Odoo Review Payslip] ◀── [Sync Payroll] ◀── [Sync Employees → Odoo] ▲ │ [HR อนุมัติทุกคำขอ] ─┘
จำไว้:
  • Supabase = source of truth (พนักงาน, เช็คอิน, OT, ลา)
  • Odoo = payroll engine (รับ push จาก HR App)
  • Hourly ต้องมีชม.ใน ledger อนุมัติแล้ว ถึง sync payroll ได้
  • Monthly ต้องมีเงินเดือนในโปรไฟล์
